# 2026-04-21 工作日志

## Bug修复：联系人私聊房间用户名映射错误
- **问题**：点击联系人跳转到消息页面后，私聊房间显示为自己的用户名而非对方的
- **根因**：
  1. 后端 `handlePrivateRoomStart` 直接返回 `room.getName()`，但私聊房间的 name 在数据库中存储的是创建者的目标用户名，第二个用户发起时返回的是自己的用户名
  2. 前端 `getDisplayRoomName` 用 `room.name`（字符串用户名）去匹配 `contact.userId`（数字 ID），永远匹配不上
  3. 前端 `handleContactClick` 不会自动选中新创建的私聊房间
- **修复**：
  - 后端：`handlePrivateRoomStart` 改用 `roomService.getPrivateRoomDisplayName()` 返回当前用户视角的正确显示名
  - 前端：简化 `getDisplayRoomName`，私聊房间直接返回 `room.name`（后端已正确设置）
  - 前端：添加 `lastPrivateRoomCreated` 通知机制，`handleContactClick` 优化为先查找已有私聊房间

## 图片预览 Bug 修复
- **问题**：双击/点击图片无法预览，只会新标签页打开
- **根因**：`FileMessage.vue` 的 `handlePreview` 使用 `window.open()`；`HomePage.vue` 的预览模态框只支持 `PendingAttachment`（待发送附件），不支持已发送消息中的图片
- **修复**：
  - `FileMessage.vue`：添加 `preview` 事件 emit，点击图片或预览按钮时发出 `{ fileName, fileSize, fileUrl, fileType }`
  - `HomePage.vue`：添加 `previewingSentImage` ref 和 `openSentImagePreview` 函数；扩展预览模态框同时支持两种预览状态；在 `FileMessage` 组件上绑定 `@preview` 事件
